Heap Queue Algorithm

The Python heapq module provides an implementation of the heap queue algorithm, a.k.a.m, the priority queue algorithm.
In Python, heaps are implemented using regular python lists. In a heap, for all k, heap[k] <= heap[2*k+1] and heap[k] <= heap[2*k+2], and this property is called heap invariant. Therefore, heap[0] is always the smallest element (i.e, it’s a min-heap), and heap.sort() maintains the heap invariant property!
To create a heap, simply use a list initialized to [], or transform a populated list into a heap via function heapify().

Primitive Numeric Types

Posted on 2019-03-20 | In Python | Comments:

Integers

Integers in Python3 are unbounded. The maximum integer representable is a function of the available memory.
There is no concept of an unsigned shift/integer in Python since integers have infinite precision. (Need to be careful about shifting here).

Floats

Floats, unlike integers, are not infinite precision. float('inf') and float('-inf') are used to refer to infinity. These values are comparable to integers and can be used to create pseudo max-int and pseudo min-int.
When comparing floating point values use math.isclose(). It is robust as for when comparing very large values, and can handle both absolute and relative differences.

Bit Manipulation

Posted on 2019-02-21 | Edited on 2019-03-20 | In Data Structures and Algorithms | Comments:

Basic Operations

Important Points

  1. A negative number -x is treated by bitwise operators as complement(x - 1).
  2. -1 is treated as “…11111110” since integers have infinite precision in Python.
  3. There is no concept of an unsigned shift in Python.
  4. ~ x is the same as - x - 1.
  5. For a negative number -x, complement(-x) = complement(complement(x - 1)) = x - 1. For a positive number x, complement(x) = - x - 1.
